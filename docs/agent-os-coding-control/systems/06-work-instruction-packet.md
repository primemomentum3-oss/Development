# 06 - Work Instruction Packet

## Plain meaning

The Work Instruction Packet is the clear job contract Agent OS gives to a coding worker before it starts.

It should be generated by Agent OS, not invented by the agent.

## What it does

It tells the worker:

- what task it is doing.
- what role it has.
- where it may work.
- what it must not touch.
- what checks are required.
- what proof is needed.
- what counts as ready.
- what still requires Stig approval.

## How it works

Agent OS builds the packet from backend records.

A simple packet includes:

```text
task goal
project and worktree
allowed paths
forbidden paths
required command
role rules
memory/context included
what to do if blocked
what done does and does not mean
```

## What it requires

- Task record.
- Validation contract.
- Worktree location.
- Role profile.
- Required checks.
- File scope rules.
- Approved project memory if relevant.
- A saved copy of the packet.

## Who owns it

Agent OS creates and stores it.

The agent reads and follows it.

Stig and future agents should be able to inspect it later.

## Terminal-based feasibility

Fully feasible with terminals.

Agent OS can write the packet to a file, send it into the terminal prompt, or both.

## Why it matters

A coding worker should not receive vague instructions.

The packet makes the job boundary clear before any code changes happen.
